responsive header: use case for native css variables

The header of this blog is a great example of when to use a native CSS variable. It's a typical responsive header in a fixed position. The value 40px jumps to 70px for a number of properties at the desktop screen size:

  • padding on the <body> to prevent visual overlap
  • negative margin on the .header also to prevent visual overlap
  • height of the .header
  • vertically centering text using line-height
:root {
	--top-space: 40px;
}

/* --- mobile first --- */
body {
	padding-top: var(--top-space);
}

body .header {
	margin-top: calc(var(--top-space) * -1);
	line-height: var(--top-space);
	height: var(--top-space);
	position: fixed;

	/* etc.. */
}

/* --- desktop --- */
@media (min-width: 993px) {
	:root {
		--top-space: 70px;   /* slam! */
	}
}

With just one property change in the media query, we've changed everything we need for desktop size. Slam!

:root is the same as <html> but wins when there's a tie. It's en vogue to define CSS variables here.

The variables feature from a CSS preprocessor like SASS could not achieve this since those variables are unchangeable after page load.
